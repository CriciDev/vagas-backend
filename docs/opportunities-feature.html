Criciuma Devs Backend

Feature de oportunidades

Esta entrega adiciona o primeiro CRUD real de oportunidades no backend, seguindo o contrato aprovado no OpenSpec e o padrão já usado no módulo de developers.

O que mudou

O backend agora tem um novo pacote internal/opportunities com modelo, validação, service, repository PostgreSQL, controller HTTP e testes.

  • Criação de oportunidades por admin.
  • Listagem pública apenas de oportunidades publicadas.
  • Busca pública por id apenas quando o status é published; admin autenticado também consegue revisar rascunhos pelo id.
  • Atualização e remoção protegidas por role admin.

Arquivos principais

  • internal/opportunities/opportunities.go: tipos, enums e erros.
  • internal/opportunities/usecase.go: regras de validação.
  • internal/opportunities/repository.go: persistência PostgreSQL.
  • internal/opportunities/controller.go: rotas HTTP.
  • internal/database/database.go: criação da tabela.
  • cmd/server/main.go: registro das rotas.

Rotas adicionadas

As rotas ficam sob /api/opportunities. Leituras são públicas; escritas exigem autenticação e role admin.

GET/api/opportunities
GET/api/opportunities/:id
POST/api/opportunities
PUT/api/opportunities/:id
DELETE/api/opportunities/:id

Modelo

Obrigatórios

title, description, organization_name, type, work_mode e pelo menos um canal de contato: contact_email ou contact_url.

Opcionais

organization_url, location, salary_range, seniority, skills, expires_at e status.

Enums aceitos

full_time part_time contract freelance volunteer project mentorship remote hybrid on_site draft published closed archived

Exemplo de payload

Na criação, quando status não é enviado, o backend assume published. Na atualização, a ausência de status preserva o status atual.

{
  "title": "Go Developer",
  "description": "Build community tools",
  "organization_name": "Criciuma Devs",
  "organization_url": "https://criciumadevs.local",
  "type": "full_time",
  "work_mode": "remote",
  "location": "Criciuma",
  "salary_range": "R$ 8k - R$ 12k",
  "seniority": "senior",
  "skills": ["Go", "PostgreSQL"],
  "contact_email": "jobs@criciumadevs.local"
}

Persistência

A tabela opportunities foi adicionada no bootstrap local do PostgreSQL com constraints para garantir contato obrigatório e valores válidos para type, work_mode e status.

Verificação

A implementação foi validada com make check, incluindo formatação, vet, testes e build do servidor.
